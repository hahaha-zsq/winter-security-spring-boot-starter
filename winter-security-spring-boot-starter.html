/ 项目简介 概述

将 Spring Security 安全认证零配置地嵌入 Spring Boot 应用的启动器，一个依赖即可获得完整的认证授权能力。

基于 Spring Boot 2.6.11 构建，专为微服务架构设计的轻量级认证授权解决方案。通过 Spring Boot Starter 机制，将复杂的安全配置简化为引入一个依赖即可完成——零配置即可获得 JWT 认证、网关集成、跨线程上下文传递等核心能力。

采用双路径认证策略——网关认证模式从 HTTP 请求头获取网关传递的用户信息（推荐生产环境使用），JWT 直连模式解析 Bearer Token 进行身份验证（适合开发调试）。两种模式自动识别，无需手动切换。同时提供 TokenAuthenticator 函数式接口作为可扩展点，支持各种 Token 类型（JWT、OAuth2、自定义 Token 等），满足不同场景的认证需求。

为什么选择 Winter Security
  • 传统 Spring Security 配置繁琐重复，每个微服务都要编写相同的安全过滤器链、认证逻辑和异常处理。
  • 本 Starter 通过 Auto-Configuration 自动化这一切，所有扩展点基于 @ConditionalOnMissingBean，提供默认实现的同时支持一键替换。
  • 基于阿里巴巴 TransmittableThreadLocal（TTL）实现跨线程上下文传递，解决线程池复用导致的上下文丢失问题。
  • 同时填充 SecurityContextHolder 和 WinterSecurityContextHolder 两套上下文，兼容现有代码的同时提供更便捷的 API。
  • 内置合理的白名单默认值，涵盖 actuator、Swagger、静态资源等 18 个路径，安全性与易用性兼顾。

/ 核心能力 特性

01 双路径认证 自动识别请求来源：若 HTTP 请求头中包含 X-User-Id 和 X-Username，则判定为网关认证模式，直接从请求头中提取用户信息。若网关信息不存在，则回退至 JWT 直连模式。两种模式对业务代码完全透明。 核心
02 零配置启动 spring.factories 自动加载 SecurityAutoConfiguration。@ConditionalOnClass 确保仅在安全环境激活，@ConditionalOnMissingBean 允许一键替换默认实现。内置 18 个白名单默认路径，启动即用。 核心
03 跨线程上下文 基于阿里巴巴 TransmittableThreadLocal（v2.14.5）实现用户上下文在线程间的自动传递。在线程池复用场景下也能正确传播上下文。请求结束后 finally 块确保彻底清除，防止数据泄漏。 核心
04 方法级权限 SecurityConfig 通过 @EnableGlobalMethodSecurity 一次性启用 @PreAuthorize（SpEL 表达式）、@Secured（角色检查）、@RolesAllowed（JSR-250 标准）三种注解。所有注解自动生效。 安全
05 双重上下文 认证成功后同时设置两套上下文：WinterSecurityContextHolder（基于 TTL 的自定义上下文）和 SecurityContextHolder（Spring Security 标准上下文）。新代码推荐 WinterSecurityContextHolder，已有代码无需改动。 基础设施
06 函数式扩展 TokenAuthenticator 为 @FunctionalInterface，只需实现 authenticate(String) 单一方法。支持 JWT、OAuth2、自定义 Token 等任意类型。返回 AuthResult 封装类，提供 success/failure 工厂方法。 扩展
07 网关请求头规范 标准化 X-User-Id、X-Username、X-User-Roles、X-User-Permissions 四个请求头。网关认证后注入，下游服务直接使用。网关必须移除客户端原始请求头防止身份伪造。 安全
08 统一异常处理 AuthenticationEntryPoint 和 AccessDeniedHandler 委托给 HandlerExceptionResolver，通过 @ControllerAdvice 统一处理。认证异常仅记录日志，不阻断过滤器链。 基础设施
09 权限缓存优化 CustomUserDetails 的 getAuthorities() 将权限字符串延迟转换为 SimpleGrantedAuthority 并缓存到 transient 字段。setPermissions() 时自动清除缓存。 性能

/ 架构设计 架构

过滤器链

每个 HTTP 请求进入后，JwtAuthenticationFilter（扩展自 OncePerRequestFilter）在 UsernamePasswordAuthenticationFilter 之前拦截。自动识别网关模式与 JWT 模式。

SecurityContextPersistenceFilter → JwtAuthenticationFilter → SecurityContextHolder → WinterSecurityCtx → FilterSecurityInterceptor

JwtAuthenticationFilter 处理流程

1请求进入，提取 Authorization 头
2白名单匹配 — 命中则跳过认证直接放行
3网关模式检测 — X-User-Id 和 X-Username 是否存在
4是 → 从请求头提取用户信息，跳过 Token 解析
5否 → JWT 模式，提取 Bearer Token 调用 TokenAuthenticator
6设置双重上下文 — SecurityContextHolder + WinterSecurityContextHolder
7finally 块清理所有上下文，防止线程池数据泄漏

自动装配流程

Spring Boot 应用 ├═ winter.security.* 配置 (application.yml) ├═ META-INF/spring.factories ├ └═ SecurityAutoConfiguration // @ConditionalOnClass ├ ├═ 条件化 Bean 注册 ├ SecurityProperties // @EnableConfigurationProperties ├ TokenAuthenticator // @ConditionalOnMissingBean → DefaultTokenAuthenticator ├ JwtAuthenticationFilter // @ConditionalOnMissingBean ├ SecurityConfig // @Import({SecurityConfig.class}) ├ └═ SecurityConfig (WebSecurityConfigurerAdapter) CSRF disable · STATELESS · whitelist · addFilterBefore · @EnableGlobalMethodSecurity

Bean 加载顺序

Spring 容器启动时，Auto-Configuration 按以下顺序注册 Bean。标记红色的可通过自定义 @Component 替代。

SecurityProperties → TokenAuthenticator → JwtAuthFilter → SecurityConfig → WinterCtxHolder → SecurityCtxHolder

/ 使用流程 工作流程

01
添加 Maven 依赖
在项目的 pom.xml 中引入以下依赖。已发布至 Maven 中央仓库，无需额外配置仓库地址。
<!-- pom.xml --> <dependency> <groupId>io.github.hahaha-zsq</groupId> <artifactId>winter-security-spring-boot-starter</artifactId> <version>0.0.3</version> </dependency>
02
实现 TokenAuthenticator 接口
生产环境必须实现。Spring Boot 通过 @ConditionalOnMissingBean 自动注入自定义 Bean，替代默认的开发用验证器。
@Component public class JwtTokenAuthenticator implements TokenAuthenticator { @Value("${jwt.secret}") private String jwtSecret; @Autowired private UserService userService; @Override public AuthResult authenticate(String token) { try { Claims claims = Jwts.parser() .setSigningKey(jwtSecret) .parseClaimsJws(token) .getBody(); Long userId = Long.valueOf(claims.getSubject()); User user = userService.getById(userId); ValidateToken result = ValidateToken.builder() .valid(true) .userId(userId) .userName(claims.get("username", String.class)) .roles(user.getRoles()) .permissions(user.getPermissions()) .build(); return AuthResult.success(result); } catch (ExpiredJwtException e) { return AuthResult.failure("Token 已过期"); } catch (SignatureException e) { return AuthResult.failure("Token 签名无效"); } catch (Exception e) { return AuthResult.failure("验证失败: " + e.getMessage()); } } }
03
配置 application.yml（可选）
自定义白名单、请求头名称等。不配置则使用合理的默认值，可直接跳过此步骤。
# application.yml winter: security: whitelist: urls: - /api/public/** - /auth/** - /actuator/** user-id-header: X-User-Id username-header: X-Username roles-header: X-User-Roles permissions-header: X-User-Permissions
04
获取用户信息与权限控制
认证成功后，两套上下文同时可用。WinterSecurityContextHolder 提供 9 个线程安全的 API 方法。
@RestController @RequestMapping("/api/user") public class UserController { @GetMapping("/info") public UserInfo getCurrentUser() { String userId = WinterSecurityContextHolder.getUserId(); return new UserInfo(userId, WinterSecurityContextHolder.getUsername(), WinterSecurityContextHolder.getRoles()); } @PreAuthorize("hasAuthority('user:delete')") @DeleteMapping("/{id}") public void deleteUser(@PathVariable Long id) { } }
05
网关集成与请求头规范
使用 Spring Cloud Gateway 时，网关过滤器中验证 Token 后将用户信息注入 HTTP 请求头。网关必须移除客户端原始传递的安全请求头以防止身份伪造。
@Component public class AuthGatewayFilter implements GlobalFilter, Ordered { @Override public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) { String token = extractToken(exchange.getRequest()); UserInfo userInfo = validateToken(token); ServerHttpRequest request = exchange.getRequest().mutate() .header("X-User-Id", userInfo.getUserId().toString()) .header("X-Username", userInfo.getUsername()) .header("X-User-Roles", String.join(",", userInfo.getRoles())) .header("X-User-Permissions", String.join(",", userInfo.getPermissions())) .build(); // 移除原始 Authorization 头 request = request.mutate() .headers(h -> h.remove("Authorization")).build(); return chain.filter( exchange.mutate().request(request).build()); } @Override public int getOrder() { return -100; } }

X-Username zhangsan 用户登录名
X-User-Roles admin,user 角色列表，逗号分隔

/ 注意事项 重要提示

网关请求头 网关必须移除客户端传递的 X-User-Id、X-Username 等请求头，防止用户伪造身份信息。网关是信任的边界。
TokenAuthenticator 必须替换 默认实现仅用于开发调试，接受 "DefaultTokenAuthenticator" 前缀 Token 并返回硬编码用户（ID=666666）。
CSRF 默认禁用 适用于无状态 REST API。若需要有状态场景需自行覆盖 SecurityConfig 的 configure(HttpSecurity) 方法。
STATELESS 会话 Session 策略为 STATELESS，不创建 HttpSession。需要会话的场景需自行调整 SessionCreationPolicy。
异步上下文传播 TTL 确保 @Async 和自定义线程池中用户上下文自动传递。需确保线程池使用 TTL 包装方式。
finally 防泄漏 过滤器同时调用 WinterSecurityContextHolder.clear() 和 SecurityContextHolder.clearContext()，避免线程池复用导致的数据污染。
异常隔离设计 认证异常仅记录日志，不阻断过滤器链。业务代码可通过检查 SecurityContext 是否为空来判断认证状态。
Jackson 安全 解析逗号分隔字符串时使用 stream().collect(Collectors.toList()) 而非 Arrays.asList()，避免 Jackson 反序列化类型白名单问题。
CustomUserDetails 权限缓存 getAuthorities() 将权限延迟转换为 SimpleGrantedAuthority 并缓存到 transient 字段。setPermissions() 时自动清除缓存。白名单内建 18 个默认路径（actuator、健康检查、静态资源、Swagger、auth 端点、错误处理等）。
